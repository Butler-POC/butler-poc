# 건물 상세 페이지 신설 + 건축물대장/삭제 이동 — 작업 보고서

- 작성일: 2026-06-03
- 작업자: StabAn-NRH
- 범위: Butler Landlord — 건물 상세 페이지(`BuildingDetailView`)

---

## 1. 배경

기존엔 건물 카드 안에 건축물대장 등록 링크와 ✕ 즉시 삭제가 섞여 있었다. 모바일에서 ✕ 오터치로 인한 사고 삭제 위험이 있고, 정보 표시도 분산돼 있어 **건물 단위 상세 페이지**로 정리했다.

## 2. 변경 내용

### 2-1. 건물 상세 페이지 신설
- 내 건물 목록(`BuildingsView`)에서 **건물 카드를 클릭하면 상세 페이지로 이동**(`/app/landlord/buildings/:id`).
- 상세 페이지는 **건축물대장(L-02) 정보를 라벨–값 행으로 정리**해 표시(대지위치·지번·명칭·주용도·주구조·대지/건축/연면적·건폐율·용적률·층수·높이·세대수·주차·사용승인일 중 값이 있는 항목만). 미등록 시 안내 문구.
- 정보 아래 **순서대로 ① 건축물대장 등록/수정 ② 건물 삭제** 버튼 배치.

### 2-2. 건축물대장 등록/수정 이동
- 건물 카드의 건축물대장 섹션·등록 링크 제거 → 상세 페이지로 이동.
- 등록 화면(`BuildingLedgerView`)의 뒤로가기·저장 후 이동을 **상세 페이지 복귀**로 변경.

### 2-3. 건물 삭제 안전장치
- 카드의 ✕ 즉시 삭제 제거(오터치 방지). 삭제는 상세 페이지에서만.
- 삭제 시 **확인 팝업**으로 재확인. 경고 문구에 "등록된 임차인·공실 등 건물과 관련된 모든 정보가 함께 삭제되며 되돌릴 수 없다"는 내용 포함(Prisma onDelete: Cascade 동작과 일치).

### 2-4. 카드 단순화
- 카드는 클릭 가능한 요약(등기 정보: 주소·명칭·용도·소유자·근저당)으로 정리, 우상단 "상세 →" 표시.

## 3. 주요 파일

- `views/landlord/BuildingDetailView.vue` (신설)
- `router/index.ts` — `landlord-building-detail` 라우트 추가
- `components/domain/BuildingCard.vue` — 카드 클릭→상세, ✕·건축물대장 섹션 제거
- `views/landlord/BuildingsView.vue` — remove 핸들러 제거
- `views/landlord/BuildingLedgerView.vue` — 복귀 경로 변경

## 4. 비고

- 삭제 재확인 팝업은 이후 공용 컴포넌트(`ConfirmDialog`)로 추출·통일됨 → 별도 보고서(Delete-UX-unify) 참고.
- 카드의 공실 건수 표시는 공실 페이지 독립화 작업에서 제거됨.
