# L-04 임차인 월세 연체 강조 표시 — 구현 보고서

> 작성일: 2026-05-31 · 작업자: StabAn-NRH
> 관련: L-03(임차인 등록). L-03에서 수집한 **월세·납부일**을 바탕으로 연체를 판정·강조한다.

---

## 1. 무엇을 만들었나

임대인이 **월세가 연체된 임차인을 한눈에** 보도록, 임차인 목록에서 연체 건을 강조하고 건물 카드에 연체 인원을 표시한다. 또한 **월세 납부 처리/취소**로 상태를 갱신할 수 있다.

핵심 과제: "연체"를 알려면 **납부 여부 상태**가 있어야 한다. L-03 `Tenant`에는 월세·납부일만 있어 납부 이력이 없으므로, **마지막 납부월(`lastPaidMonth`, YYYY-MM)** 한 필드를 추가하고, 서버가 오늘 날짜·납부일·마지막 납부월로 상태를 계산한다.

---

## 2. 연체 판정 로직 (서버 계산)

`rentState(tenant, now)` 가 4가지 상태를 반환한다.

| 상태 | 조건 | 표시 |
|---|---|---|
| `NO_RENT` | 월세 없음(전세 등, monthlyRent ≤ 0) | 표시 안 함 |
| `UNKNOWN` | 월세 있음 + 납부 기록 없음(lastPaidMonth null) | "납부 정보 없음" (강조 X — 신규 등록 오탐 방지) |
| `PAID` | lastPaidMonth ≥ 납부기준월 | "납부 완료" (초록) |
| `OVERDUE` | lastPaidMonth < 납부기준월 | **"연체 N개월" (빨강 강조)** |

- **납부기준월(dueMonth)**: 오늘이 `납부일` 이후면 이번 달, 전이면 직전 달. (예: 납부일 25일, 오늘 5/20 → 기준월 4월)
- **연체 개월수**: `dueMonth − lastPaidMonth`(개월). 예) 최종납부 3월, 기준월 5월 → 2개월.
- 신규 등록(UNKNOWN)은 일부러 연체로 강조하지 않는다 — 납부 이력 미상이라 오탐이 되기 때문.

---

## 3. 조작 (납부 처리/취소)

- **납부 처리**: `lastPaidMonth = dueMonth` 로 설정 → PAID.
- **납부 취소**: `lastPaidMonth = dueMonth−1` 로 설정 → 다시 OVERDUE(1개월).
- 클라이언트는 응답의 `dueMonth` 를 그대로 사용해 서버에 전달(`PATCH .../rent`).

---

## 4. API

| 메서드 | 경로 | 설명 |
|---|---|---|
| GET | `/api/buildings/:id/tenants` | 임차인 목록 — 각 항목에 **`rentStatus`** 동봉(서버 계산) |
| PATCH | `/api/buildings/:id/tenants/:tenantId/rent` | 마지막 납부월 설정(`{ lastPaidMonth: "YYYY-MM" \| null }`) |
| GET | `/api/buildings` | 건물 목록 — 각 건물에 **`overdueTenants`**(연체 인원) 동봉 |

- 모두 임대인 전용 + 본인 건물 소유 검증(기존 가드 공유).
- `lastPaidMonth` 형식 검증(`YYYY-MM`), 위반 시 400.

## 7. 한계 · 후속

- 납부 이력은 **마지막 납부월 1개 값**으로만 관리(개별 월 납부내역 테이블 아님). 부분 납부·정확한 회계는 범위 외.
- 기준월 계산은 **서버 현재 시각** 기준. 타임존은 서버 로컬을 따른다.
- 자동 알림(연체 발생 시 푸시/메일)은 미구현 — 후속 가능.
- 전세(`NO_RENT`)는 연체 개념이 없어 제외.
