# L-01 내 건물 등록 — 구현 보고서

> 작성일: 2026-05-30 · 작업자: StabAn-NRH
> 기능: 등기부등본을 OCR 스캔하거나 직접 입력해 임대인의 건물 정보를 등록한다. (임대인 전용)

---

## 1. 무엇을 만들었나

임대인(Butler Landlord)이 자기 건물을 등록·조회·삭제하는 기능. 등록은 두 방식:

1. **등기부등본 스캔** — 문서를 올리면 C-01 OCR로 텍스트를 뽑고, **LLM이 핵심 필드를 구조화**해 폼을 자동으로 채운다. 사용자가 확인·수정 후 저장.
2. **직접 입력** — 빈 폼에 수동 입력.

추출/입력 필드: 소재지(필수)·건물명칭·구조·용도·연면적(㎡)·층수·소유자·지분·등기고유번호.

---

## 2. 동작 흐름

```
[스캔 등록]
  파일 업로드 → POST /api/buildings/parse
     → C-01 scanFile (PDF 텍스트추출 또는 비전 OCR)
     → LLM 구조화 추출(extractJSON + 등기부 프롬프트)
     → { fields, rawText, ocrSource } 반환  ※ 저장 아직 안 함
  폼 자동완성 → 사용자 확인/수정
     → POST /api/buildings (source: "ocr", rawText 포함) → 저장

[수동 등록]
  폼 입력 → POST /api/buildings (source: "manual")
```

저장 전에 파싱 결과를 사용자가 검토하는 2단계 구조라, OCR 오인식을 사람이 교정할 수 있다.

---

## 3. API (모두 임대인 전용 — requireAuth + requireRole('LANDLORD'))

| 메서드 | 경로 | 설명 |
|---|---|---|
| `POST` | `/api/buildings/parse` | 등기부 업로드 → OCR + 구조화 추출 (저장 X) |
| `POST` | `/api/buildings` | 건물 등록 |
| `GET` | `/api/buildings` | 내 건물 목록 |
| `GET` | `/api/buildings/:id` | 단건 조회 |
| `DELETE` | `/api/buildings/:id` | 삭제 |

- 다른 역할(임차인·중개업자)이 호출하면 **403**.
- 모든 조회/삭제는 **본인 소유(ownerId)** 로 스코프.

---

## 4. 데이터 모델 (`buildings` 테이블)

`Building`(ownerId→User, onDelete: Cascade) — address(필수) 외 등기 필드 전부 nullable, `source`("manual"|"ocr"), `rawText`(OCR 원문, @db.Text). User에 `buildings Building[]` 관계 추가.

## 7. 한계 · 후속 과제

- **주소 정규화/검증 없음** — 도로명주소 API 연동(추후) 시 정확도 향상.
- **수정(편집) 기능 미구현** — 현재 등록·삭제만. PUT 추가 필요.
- **등기 권리관계(갑구/을구) 미수집** — 근저당 등은 범위 외(필요 시 확장).
- **이미지 다중 페이지** — 현재 단일 파일. 여러 장 등기부는 추후.
- **L-02(건축물대장)** 와 필드 일부 중복 — 모델 통합/분리 정책은 L-02 착수 시 결정.
