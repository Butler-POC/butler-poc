# 간단 법률 상담 구현 보고서 — L-06

- **작성일**: 2026-06-01
- **작업자**: jh
- **범위**: L-06 간단 법률 상담 (버틀러 오너) — 본인 건물·임차인 맥락을 자동 주입한 LLM 법률 챗봇

---

## 1. 무엇을 / 왜

임대인이 **별도 컨텍스트 입력 없이** 자기 건물·임차인 상황을 반영한 임대차 법률 상담을 받게 했다. 사용자가 일일이 상황을 설명하지 않아도, 서버가 로그인 임대인의 데이터를 시스템 프롬프트로 합성해 LLM에 전달한다.

## 2. 어떻게

### 서버
- **`prompts/legal.prompt.ts`** (신규)
  - `LegalContext` 타입(건물 + 임차인 요약: 보증금·월세·계약기간·연체·특약) 정의.
  - `buildLegalSystemPrompt(ctx)` — 관리 현황을 텍스트로 렌더 + 응답 규칙(합니다체, 호들갑·이모지 금지, ₩ 표기, 전문가 상담 권유, 참고용 고지)을 담은 시스템 프롬프트 생성.
  - `LEGAL_DISCLAIMER = '참고용 · 법적 효력 없음'` 상수 export(문구 일치용).
- **`controllers/chat.controller.ts`** (신규) — `legalChat` 핸들러
  - `req.user`(임대인) → `prisma.building`(ownerId, `include: tenants`)로 본인 데이터 조회 → `LegalContext` 합성.
  - 기존 **`llm.service.chat({ system, messages })`** 호출(단일 LLM, 프롬프트 분기). 히스토리는 최근 10턴만 전달.
  - 응답: `{ reply, disclaimer }`.

### 클라이언트
- **`views/landlord/LegalChat.vue`** (신규)
  - 요청-응답형 채팅 UI(소켓 아님). `@/api/client`로 `POST /api/chat { mode:'legal', message, history }`.
  - **디스클레이머 배너 상시 노출**(amber inline 배너, Design.md BANNER 규격) — "참고용 · 법적 효력 없음".
  - 빈 상태: brass 점 + "등록하신 건물·임차인 정보를 바탕으로 답합니다." 안내. 합니다체, 이모지 없음.

## 3. 단일 LLM / 프롬프트 분기 원칙 준수
- 새 AI 제공자/SDK를 추가하지 않았다. **기존 `llm.service` 하나를 호출**하고, 법률 모드는 `legal.prompt.ts`로만 분기한다.
- 채팅 컨트롤러는 향후 T-02(하자 상담)용 `repairChat`도 같은 파일에 추가할 구조로 열어 두었다.

## 4. 디스클레이머 이중화
- **프롬프트 차원**: 시스템 프롬프트에 "참고용·법적 효력 없음 인지" + 전문가 상담 권유 명시.
- **화면 차원**: 상단 amber 배너로 상시 노출. (설계 규칙: 법률 AI 출력에 디스클레이머 필수)

## 5. 가정한 기반 인터페이스 (머지 시 정합)

| 대상 | 가정 |
|---|---|
| `services/llm.service` | `chat({ system: string, messages: {role,content}[] }) => Promise<string>`. .env `API_KEY`(Claude) 내부 사용 |
| `lib/prisma` | default export. `prisma.building`(ownerId) + `building.tenants` 관계 |
| `Tenant` 필드 | name/unit/deposit/monthlyRent/(leaseStart·leaseEnd 또는 leasePeriod)/overdue(또는 isOverdue)/specialTerms — 필드명 미확정 대비 옵셔널 체이닝 처리 |
| `api/client` | axios 인스턴스(baseURL `/api`) default export |
| `req.user` | `{ id, name? }` (auth 미들웨어 주입) |
