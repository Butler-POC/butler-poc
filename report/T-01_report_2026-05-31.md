# T-01 임차 건물 등록 — 구현 보고서

> 작성일: 2026-05-31 · 작업자: StabAn-NRH
> 기능: 임차인이 자신이 임차한 건물(임대차계약) 정보를 OCR 스캔 또는 수동 입력으로 등록한다. (**임차인 전용**)

---

## 1. 무엇을 만들었나

임차인(Butler Tenant)이 본인 임대차계약 정보를 등록·조회·삭제하는 기능. 등록은 두 방식:

1. **계약서 스캔** — 임대차계약서를 올리면 LLM 비전이 임차인 관점 필드를 구조화해 폼을 자동으로 채운다. 확인·수정 후 저장.
2. **직접 입력** — 빈 폼에 수동 입력.

필드: 임차 건물 주소(필수)·건물명칭·동/호·임대인·보증금·월세·관리비·납부일·계약기간·계약일·특약사항.

---

## 2. 설계 판단 — L-03과의 관계

L-03(임대인의 "임차인 등록")과 **같은 임대차계약서**를 다루지만, 데이터 주체가 다르다.

| | L-03 Tenant | T-01 Lease |
|---|---|---|
| 소유 | 임대인이 등록, **Building 에 종속(buildingId)** | 임차인 본인, **User 에 종속(userId)** |
| 건물 | 이미 등록된 건물 | 건물 FK 없음 → **주소를 직접 보유** |
| 추출 관점 | 임차인 이름·연락처 중심 | **목적물 주소·임대인·계약조건** 중심 |

그래서 **별도 `Lease` 모델 + 임차인 관점 추출 프롬프트**로 분리했다. (비전 추출 인프라 `extractJSONFromMedia` 는 공유)

---

## 3. 데이터 모델 — `Lease` (User 1:N)

`address`(필수) 외 전부 nullable. `userId → User`(onDelete: Cascade).

| 필드 | 의미 |
|---|---|
| address | 임차 건물(목적물) 주소 (필수) |
| buildingName / unit | 건물 명칭 / 동·호 |
| lessorName | 임대인(소유자) |
| deposit / monthlyRent / maintenanceFee | 보증금 / 월세 / 관리비(원) |
| paymentDay | 월세 납부일(1~31) |
| leaseStart / leaseEnd / contractDate | 계약 시작·종료·체결일 |
| specialTerms | 특약사항(@db.Text) — **T-02 하자 상담용** |
| source / rawText | "manual"\|"ocr" / OCR 원문 |

`User.leases Lease[]` 관계 추가.

## 4. API (모두 임차인 전용 — requireAuth + requireRole('TENANT'))

| 메서드 | 경로 | 설명 |
|---|---|---|
| POST | `/api/leases/parse` | 계약서 업로드 → 비전 구조화 추출 (저장 X) |
| GET | `/api/leases` | 내 임차 건물 목록 |
| POST | `/api/leases` | 임차 건물 등록 |
| GET | `/api/leases/:id` | 단건 조회 |
| DELETE | `/api/leases/:id` | 삭제 |

- 다른 역할(임대인·중개업자)이 호출하면 **403**.
- 모든 조회/삭제는 **본인(userId)** 으로 스코프.
- 이미지 업로드는 클라(`checkUploadSize` 7.5MB)·서버(`llm.service` 413 가드) 양쪽에서 크기 사전 차단.

## 7. 한계 · 후속

- 임차 건물 **수정(편집)** 미구현 — 등록·삭제만.
- 등록된 `specialTerms` 는 **T-02 하자 상담**의 컨텍스트로 사용 예정.
- 임차인의 임대차와 임대인의 Building/Tenant 간 **자동 매칭은 없음**(각자 독립 등록). 추후 주소 기반 연결 검토 가능.
