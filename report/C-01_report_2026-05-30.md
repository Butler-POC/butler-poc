# C-01 OCR 스캐너 — 구현 보고서

> 작성일: 2026-05-30 · 작업자: StabAn-NRH
> 기능: PDF·이미지 파일에서 텍스트를 추출한다. (공통 기능, L-01·L-02·L-03·T-01 에서 재사용 예정)

---

## 1. 무엇을 만들었나

문서(PDF·이미지)를 업로드하면 텍스트를 추출해 돌려주는 기능을 **서버 API + 클라이언트 화면**으로 완성했습니다.
프로젝트가 빈 스캐폴드 상태였기 때문에, C-01을 동작시키기 위한 **서버(Express)·클라이언트(Vue) 최소 기반**도 함께 세웠습니다.

핵심 아이디어는 **파일 종류로 처리를 분기하는 하이브리드**입니다.

```
업로드 파일
 ├─ PDF  → pdf-parse 로 텍스트 레이어 직접 추출 (무료·정확·빠름)
 │         ├─ 텍스트 충분        → 그대로 반환            (source: "pdf-text")
 │         └─ 텍스트 거의 없음    → 스캔본으로 보고 ↓ 폴백
 └─ 이미지(또는 스캔 PDF) → Claude 비전으로 OCR          (source: "vision")
```

- 인터넷등기소 등에서 발급한 **디지털 PDF**(텍스트 레이어 있음)는 LLM 호출 없이 무료로 추출합니다.
- 텍스트 레이어가 없는 **스캔 PDF·사진**만 Claude 비전 API로 넘겨 비용을 최소화합니다.
- PDF 스캔본은 이미지로 변환하지 않고 **PDF 원본 그대로** Claude `document` 블록으로 전송합니다.

## 2. 동작 방식 (요청 흐름)

```
[클라이언트]  파일 선택 → POST /api/ocr/scan?category=...  (multipart, field: file)
      │
      ▼
[upload]   multer 디스크 저장 → server/uploads/<category>/<timestamp>-<name>
      │
      ▼
[ocr.service]  PDF? → pdf-parse 시도 → (부족하면) llm.service 비전 폴백
               이미지? → llm.service 비전
      │
      ▼
[응답]  { fileName, mimeType, sizeBytes, storedPath, text, source, provider, charCount }
```

### API

| 메서드 | 경로 | 설명 |
|---|---|---|
| `POST` | `/api/ocr/scan?category=registry\|contracts\|buildings\|misc` | 파일(field `file`) 업로드 → 텍스트 추출 |
| `GET` | `/api/ocr/health` | 비전 활성 여부 확인 `{ ok, visionEnabled }` |

> **category는 쿼리 파라미터로 전달합니다.** multer는 파일보다 앞에 온 텍스트 필드만 저장 콜백 시점에 인식하므로, 폼 필드 순서에 의존하지 않도록 쿼리로 받습니다.

### 응답 예시 (디지털 PDF)

```json
{
  "fileName": "등기부등본.pdf",
  "mimeType": "application/pdf",
  "sizeBytes": 689,
  "storedPath": "/uploads/registry/1780103410950-581937-등기부등본.pdf",
  "text": "...추출된 텍스트...",
  "source": "pdf-text",
  "provider": null,
  "charCount": 86
}
```

## 4. 환경 변수 (`server/.env`)

| 키 | 필수 | 기본값 | 설명 |
|---|:---:|---|---|
| `API_KEY` | 비전 사용 시 | — | Claude(Anthropic) API 키. 없으면 PDF 텍스트 추출만 동작 |
| `LLM_MODEL` | 선택 | `claude-sonnet-4-6` | 비전·챗봇에 쓸 모델 |
| `PORT` | 선택 | `3000` | 서버 포트 |

> `API_KEY` 가 없으면 `/api/ocr/health` 의 `visionEnabled` 가 `false` 가 되고,
> 이미지·스캔본 업로드 시 503 으로 "비전 비활성" 안내가 반환됩니다. (디지털 PDF는 정상 동작)

## 5. 실행 방법

```bash
# 최초 1회 — 루트·서버·클라이언트 의존성 설치
npm run install:all

# 개발 서버 동시 실행 (server:3000, client:5173)
npm run dev
```

브라우저에서 `http://localhost:5173` → `/scan` 화면에서 문서를 올려 추출합니다.

## 7. 알려진 한계 · 후속 과제

- **비전 경로 실측 미완** — 실제 이미지/스캔 PDF 1건으로 정확도·응답시간 확인 필요.
- **다중 페이지 PDF** — pdf-parse 는 전체 페이지 텍스트를 합쳐 반환. 비전 경로는 모델 토큰 한도 내에서 처리(현재 `max_tokens: 4096`).
- **구조화 파싱은 범위 외** — C-01 은 "원시 텍스트 추출"까지만 담당. 등기부/계약서의 필드 추출(소유자·보증금·특약 등)은 L-01·L-03 에서 이 텍스트를 입력으로 구현 예정.
- **업로드 정리 정책 없음** — `server/uploads/` 누적. 추후 보존기간·삭제 정책 필요.
- **인증 미적용** — 현재 누구나 호출 가능. 로그인(auth) 붙은 뒤 미들웨어 연결 필요.
