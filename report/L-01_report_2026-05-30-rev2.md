## 1. 추가된 것

등기사항증명서(등기부등본) **을구의 근저당권**을 추출·저장하도록 확장했다.

핵심 도메인 규칙(사용자 제공):
- 등기사항증명서 = 표제부 / 갑구(소유권) / 을구(소유권 외 권리).
- 근저당권은 **을구**에 있다.
- **실선(밑줄·취소선)이 그어진 항목 = 말소(현재 무효)** → 제외.
- 밑줄 없는 항목만 현재 유효한 근저당권 → 추출.
- 을구가 비었거나 전부 말소면 → 추출할 근저당권 없음(빈 목록).

---

## 2. 중요한 설계 변경 — 텍스트 OCR → **비전 분석**

> **말소선(밑줄)은 시각 정보**다. PDF 텍스트 레이어(pdf-parse)에는 선이 없어, 텍스트만으로는 어떤 항목이 말소됐는지 알 수 없다.

그래서 등기부 파싱을 **단일 비전 호출**로 바꿨다. 문서(이미지/PDF)를 Claude 비전에 직접 넘겨, 모델이 **밑줄을 시각적으로 보고** 말소 여부를 판별한 뒤 구조화 JSON(근저당권 배열 포함)을 반환한다.

- 신규: `llm.service.extractJSONFromMedia(systemPrompt, base64, mediaType)` — 이미지/PDF를 보고 JSON 추출.
- `prompts/registry.prompt.ts` — 말소 판별 규칙 + 을구 근저당권 스키마를 명시.
- `POST /api/buildings/parse` — 기존 `scanFile(OCR 텍스트)+extractJSON(텍스트)` → **`parseRegistryFromMedia`(비전)** 로 교체.

(참고: 공통 C-01 OCR 스캐너는 그대로 텍스트 추출용으로 유지. 등기부 "구조화+말소 판별"만 비전 경로로 분리.)

## 6. 한계 · 후속

- 말소 판별 정확도는 문서 스캔 품질·해상도에 의존. 저화질 스캔은 오판 가능 → 사용자 검토(현재 저장 전 확인 단계 있음)로 보완.
- 채권최고액은 Float 저장(원 단위 정수). 초대형 금액 정밀도 필요 시 BigInt 검토.
- 근저당권 외 을구 권리(전세권·지상권 등)는 범위 외.
