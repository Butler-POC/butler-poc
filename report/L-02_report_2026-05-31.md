# L-02 내 건물 정보 등록 (건축물대장) — 구현 보고서

> 작성일: 2026-05-31 · 작업자: StabAn-NRH
> 관련: L-01(내 건물 등록=등기부등본). 본 기능은 L-01로 등록한 건물에 **건축물대장** 정보를 덧붙인다.

---

## 1. 무엇을 만들었나

임대인이 **건축물대장**(건물의 물리적 현황 — 대지·면적·구조·용도·층수·사용승인 등)을
**OCR 스캔 또는 수동 입력**으로 등록하는 기능이다.

핵심 설계 판단:
- 건축물대장은 독립 건물이 아니라 **이미 등록된 건물(L-01)의 부가 정보**다.
  따라서 새 엔티티 `BuildingLedger`를 만들어 **Building 과 1:1로 연결**했다. (건물당 대장 1건)
- 등록 진입점은 **건물 카드(BuildingCard)** → "건축물대장 등록/수정" 링크.
- 추출·확인·저장 흐름은 L-01에서 검증된 **비전 구조화 추출 패턴을 그대로 재사용**한다.

---

## 2. 흐름

```
내 건물 목록 → [건물 카드] "＋ 건축물대장 등록"
   → L-02 화면 (/app/landlord/buildings/:id/ledger)
      ├ 대장 스캔  : PDF·이미지 업로드 → 비전 구조화 추출 → 폼 자동완성
      └ 직접 입력  : 폼에 수기 입력
   → 확인·수정 → 저장(upsert) → 목록으로
```

- 이미 대장이 있는 건물은 화면 진입 시 **수정 모드**로 기존 값이 채워진다.
- 신규일 때는 등기부의 주소·명칭을 대지위치/명칭 기본값으로 미리 채워 입력을 줄인다.

---

## 3. 데이터 모델 — `BuildingLedger` (1:1)

| 필드 | 의미 | 타입 |
|---|---|---|
| siteAddress | 대지위치 | String? |
| lotNumber | 지번 | String? |
| buildingName | 건축물 명칭 | String? |
| landArea | 대지면적(㎡) | Float? |
| buildingArea | 건축면적(㎡) | Float? |
| totalFloorArea | 연면적(㎡) | Float? |
| buildingCoverageRatio | 건폐율(%) | Float? |
| floorAreaRatio | 용적률(%) | Float? |
| mainStructure | 주구조 | String? |
| mainUsage | 주용도 | String? |
| floorsAbove / floorsBelow | 지상/지하 층수 | Int? |
| height | 높이(m) | Float? |
| households | 세대/가구/호 (표기 다양) | String? |
| parking | 주차 (표기 다양) | String? |
| approvalDate | 사용승인일 | String? |

- `buildingId @unique` + `Building.ledger BuildingLedger?` → 1:1.
- `onDelete: Cascade` — 건물 삭제 시 대장도 함께 삭제.
- 표기가 다양한 값(세대/주차/사용승인일)은 무리한 숫자화 대신 **문자열로 보존**.

---

## 4. API

| 메서드 | 경로 | 설명 |
|---|---|---|
| POST | `/api/buildings/ledger/parse` | 건축물대장 업로드 → 비전 구조화 추출 (저장 X) |
| PUT | `/api/buildings/:id/ledger` | 건축물대장 등록/수정 (건물당 1건, **upsert**) |

- 둘 다 임대인 전용(`requireRole('LANDLORD')`). PUT은 **본인 건물 소유 확인** 후 처리(아니면 404).
- 조회는 별도 엔드포인트 없이 `GET /buildings`·`GET /buildings/:id`에 `ledger`를 **include**하여 함께 내려준다.

## 7. 한계 · 후속

- 비전 추출 정확도는 스캔 품질·해상도에 의존. 저화질 문서는 일부 항목이 비어 추출될 수 있음(수동 보정 가능).
- 면적·비율은 Float 저장. 집합건축물대장의 **호별 전유/공용면적 명세**는 범위 외(요약 필드만).
- 대장과 등기부의 연면적이 다를 수 있음 — 현재는 각각 보존하며 별도 정합성 검증은 하지 않음.
