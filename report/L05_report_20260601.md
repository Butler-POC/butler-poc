---

## 1. 무엇을 / 왜

임대인이 건물 관리에 필요한 업체(**보일러 / 에어컨 / 입주청소 / 하수도**)를 건물 위치 기준으로 조회한다. 카테고리를 고르면 해당 건물 주변 업체를 **리스트 + 지도 마커**로 보여준다.

## 2. 어떻게 — 서버

- **`services/kakao.service.ts`** — 카카오 **로컬 키워드 검색**(`/v2/local/search/keyword.json`) 호출.
  - `.env KAKAO_REST_KEY`(서버 전용 REST 키)로 `Authorization: KakaoAK …` 인증.
  - 분류 → 키워드 매핑: boiler→"보일러 수리", aircon→"에어컨 설치 수리", cleaning→"입주청소", plumbing→"하수구 정비".
  - `x=lng, y=lat, radius, sort=distance, size`로 거리순 검색 → `Vendor[]`로 가공.
  - 런타임: Node 18+ 전역 `fetch` 사용.
- **`controllers/vendor.controller.ts` + `routes/vendors.ts`** — `GET /api/vendors?category=&lat=&lng=&radius=`. 카테고리·좌표 검증, `requireAuth`.

### [자료조사 메모]
지시서 `[자료조사 필요]` 항목(정확한 Kakao 카테고리 코드)은, 카테고리 그룹 코드(CT1 등)가 수선 업종을 정확히 못 잡아 **키워드 검색으로 대체**했다. 키워드 문자열은 `CATEGORY_KEYWORD` 상수에서 조정 가능.

## 3. 어떻게 — 클라이언트

- **`composables/useKakaoMap.ts`** — `.env VITE_KAKAO_JS_KEY`(클라 JS 키)로 카카오 지도 SDK를 **동적 로드(싱글턴)**. `init/setCenter/renderVendors/clearMarkers` 제공. 마커 클릭 시 InfoWindow.
- **`components/domain/VendorMap.vue`** — 지도 컨테이너 + 마커 렌더, `vendors`/`center` props 반응.
- **`api/vendors.ts`** — `fetchVendors({ category, lat, lng })`.
- **`views/landlord/VendorSearch.vue`** — 건물 선택 + 카테고리 칩(pill) + 지도 + 리스트(이름·거리·주소·전화·상세). 빈/로딩/에러 상태 처리, 합니다체.

## 4. 키 분리 준수
- **REST 키 = 서버**(`kakao.service.ts`, 비밀), **JS 키 = 클라**(`useKakaoMap`, 브라우저 노출 전제). 둘을 섞지 않음.
- REST 키는 `server/.env`, JS 키는 `client/.env`(VITE_ 접두사). 둘 다 `.gitignore`로 미추적.

## 5. 실동작 검증 (이번에 가능)
REST 키 보유로 **서버 검색은 실제 호출로 검증 완료**:
- 동일 요청(`query=보일러 수리`, 서울시청 좌표, KakaoAK 헤더)으로 카카오 정상 200 응답 확인.
- 응답 필드(`place_name/phone/road_address_name/x/y/distance/place_url/id`)가 서비스 매핑과 일치.
- → `kakao.service.searchVendors` 의 요청 구성·필드 매핑이 올바름을 확인.
- **클라 지도/마커**는 브라우저 SDK 로드가 필요해 기반(라우팅·빌드) 머지 후 확인.
