# 임대인 메인화면(오늘의 전달사항) + 기능 드로어 — 작업 보고서

- 범위: Butler Landlord 앱 — ① 좌측 기능 드로어, ② 로그인 후 메인화면(오늘의 전달사항)

## 1. 좌측 기능 드로어

- 상단바(`TopBar`) **좌상단에 햄버거 버튼** 추가 → 누르면 좌측에서 드로어가 슬라이드로 열림.
- 드로어(`NavDrawer.vue`)에서 **임대인 기능(L-) 전체 + 공통(C-) + 로그아웃**에 접근. 메인화면 바로가기 포함.
- 배경(backdrop) 클릭·라우트 이동 시 자동으로 닫힘. `PhoneFrame` 내부에 절대배치되어 폰 프레임 안에서 동작.
- 기능 목록은 `lib/roleFeatures.ts`로 분리해 홈 화면(`RoleHome`)과 공유(중복 제거). 드로어/햄버거는 임차인·중개업자 역할에도 그대로 동작.
- 열림 상태는 `stores/ui.ts`(Pinia)에서 관리.

## 2. 메인화면 — 오늘의 전달사항

로그인한 임대인이 진입하는 화면(`views/landlord/LandlordHome.vue`). 내 건물·세입자·임대차계약·공실을 종합해 **오늘 챙겨야 할 일**을 리스트로 제시한다.

### 전달사항 4종과 우선순위

| 순위 | 종류 | 조건 | 표시 정보 |
|---|---|---|---|
| 1 | 오늘 월세 납부 | 오늘이 납부일 & 이번 달 미납 | 월세·납부일 |
| 2 | 계약 만료 임박 | 만료까지 1개월 이내(아직 만료 전) | **만료까지 남은 일수** |
| 3 | 월세 연체 | 지난 달(들) 미납 | **연체 개월수·연체금액(월세 기준 N개월분)** |
| 4 | 공실 미처리 | 계약 만료로 공실이 됐으나 공실등록·묵시적갱신 안 함 | 이전 만료일 + 처리 버튼 |

- **정렬**: 우선순위 → 조건이 처음 충족된 날(먼저 등록된 사항 우선) → 건물/세대/세입자 이름.
- **헤드라인**: 화면 최상단에 `{이름} 님,` + 리스트 1순위 사항을 큰 글씨로. 헤드라인에 띄워도 **리스트에서 빠지지 않음**(별도 표출). 전달사항이 없으면 "전달드릴 사항이 없어요" 안내.

### 삭제(사용자 dismiss) 규칙

- 사용자가 각 항목을 **✕로 삭제** 가능. 삭제 기록은 서버(`DigestDismissal`)에 영속.
- **영구 숨김**: 만료 임박 / 공실 미처리 / 해당 월의 월세 납부일 → 삭제하면 다음날에도 조건 충족해도 안 보임.
- **월(月) 단위 리셋**: "오늘 월세 납부일"은 `대상+해당월(YYYY-MM)` 키로 식별 → 이번 달만 숨김, 다음 달 납부 건은 **새 전달사항**.
- **연체 예외**: 연체는 중요 사항이라 삭제해도 **다음날 다시 노출**(당일 한정 숨김).

### 공실 미처리 항목의 처리

- `[공실로 등록]` → 해당 건물 공실 등록 화면으로 이동하며 동/호 자동 입력(`?unit=`). 공실이 등록되면(동·호 매칭) 해당 항목은 자동으로 사라짐.
- `[묵시적 갱신]` → `Tenant.renewedImplicitly = true`. 이후 그 세입자의 **계약기간 관련(만료임박·공실) 전달을 중단**.

## 4. 검증

- 전달사항 계산 로직 단위 검증 **15/15 통과**(납부일 vs 연체 경계, 3개월 미납+오늘 납부일 동시, 만료창·말일 클램핑, 공실/묵시갱신 억제, 전세 제외 등).
- server(tsc)·client(vue-tsc + vite) **빌드 통과**.

## 5. 결정 사항(사용자 합의)

- 묵시적 갱신은 Tenant 플래그로 기록, 참이면 계약기간 전달 중단.
- "오늘 월세 납부일"은 월 단위 별건, 연체는 삭제해도 다음날 재노출.
- 정렬 기준 "등록일" = 조건이 처음 충족된 날(파생 계산).

## 6. 후속(미구현)

- 임차인 / 중개업자 메인화면 동일 패턴 적용 예정.
- 동/호 표기 편차가 크면 공실↔세입자 매칭(문자열 정규화 비교)이 어긋날 수 있음 — 향후 세대 식별자 정규화 고려.
