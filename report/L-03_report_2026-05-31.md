# L-03 임차인 등록 (임대차계약서) — 구현 보고서

> 작성일: 2026-05-31 · 작업자: StabAn-NRH
> 관련: L-01(내 건물 등록). 임차인은 **L-01로 등록한 건물에 속하는 임대차계약**으로 관리한다.

---

## 1. 무엇을 만들었나

임대인이 자기 건물에 대한 **임차인(임대차계약서) 정보**를 **OCR 스캔 또는 수동 입력**으로
등록·조회·삭제하는 기능이다.

설계 판단:
- 임차인은 **건물(Building)에 종속되는 1:N** 관계(`Building.tenants`).
- 진입점은 **건물 카드**의 "임차인 N명 · 관리" 링크 → 건물별 임차인 화면.
- 추출·확인·저장 흐름은 L-01·L-02에서 검증된 **비전 구조화 추출 패턴을 그대로 재사용**한다.
- **월세 납부일·특약사항** 필드를 함께 수집한다 — 각각 **L-04(월세 연체 강조)**, **T-02(하자 상담)** 의 입력이 된다.

---

## 2. 흐름

```
내 건물 목록 → [건물 카드] "임차인 N명 · 관리"
   → L-03 화면 (/app/landlord/buildings/:id/tenants)
      ├ 임차인 목록(보증금·월세·납부일·계약기간·특약)
      └ ＋ 임차인 추가
          ├ 계약서 스캔 : PDF·이미지 → 비전 추출 → 폼 자동완성
          └ 직접 입력
       → 확인·수정 → 등록 → 목록 갱신
```

---

## 3. 데이터 모델 — `Tenant` (Building 1:N)

| 필드 | 의미 |
|---|---|
| name(필수) | 임차인 이름 |
| contact | 연락처 |
| unit | 동/호 |
| lessorName | 임대인(계약상) |
| deposit | 보증금(원) |
| monthlyRent | 월세(원) |
| maintenanceFee | 관리비(원) |
| paymentDay | 월세 납부일(1~31) — **L-04 연체 판정용** |
| leaseStart / leaseEnd | 계약 시작/종료일 |
| contractDate | 계약 체결일 |
| specialTerms | 특약사항(@db.Text) — **T-02 하자 상담용** |
| source / rawText | "manual"\|"ocr" / OCR 원문 |

- `Building.tenants Tenant[]`, `onDelete: Cascade`(건물 삭제 시 임차인도 삭제).
- 건물 목록(`GET /buildings`)에 `_count.tenants`를 포함해 카드에서 임차인 수를 바로 표시.

---

## 4. API (모두 임대인 전용, 본인 건물 소유 검증)

| 메서드 | 경로 | 설명 |
|---|---|---|
| POST | `/api/buildings/tenants/parse` | 임대차계약서 업로드 → 비전 구조화 추출 (저장 X) |
| GET | `/api/buildings/:id/tenants` | 건물의 임차인 목록 |
| POST | `/api/buildings/:id/tenants` | 임차인 등록 |
| DELETE | `/api/buildings/:id/tenants/:tenantId` | 임차인 삭제 |

- 기존 buildings 라우터(이미 `requireRole('LANDLORD')`)에 추가해 역할 가드를 공유.
- 모든 임차인 작업은 대상 건물의 **소유자(ownerId) 확인** 후 수행(아니면 404).
